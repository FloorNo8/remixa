# DISCOVER — S: Scope the Domain

**Remixa ("EU TikTok Sound Lab")** is an EU-compliance-first AI music-generation platform built by Floor No 8 SRL. Users generate short (≈15s) AI music tracks (via Replicate/MusicGen), remix existing tracks to form a *lineage graph*, and earn automatic royalties that flow up that remix lineage (parent €0.07, or parent €0.05 + grandparent €0.02 on 3-level chains, with a €0.03 platform fee). The product's defensibility thesis (its "L5 corner") is that the remix-royalty lineage graph is the single source of truth binding four systems together: **provenance** (C2PA manifests), **royalties** (the split engine), **discovery** (remix-chain visualization), and **network effect** (earnings tied to the graph). Compliance surface is deliberately heavy: C2PA content credentials, GDPR erasure with multi-hop royalty survival, EU AI Act Art. 53 disclosure, DSA, and VAT MOSS.

**Stack:** FastAPI (Python 3.11) backend on Fly.io (`eu-sound-lab`, region ams, Fly Postgres + Redis); Next.js 14 frontend on Vercel with Clerk auth and Stripe payments. ~15.3K LOC Python backend, ~7.7K LOC TypeScript frontend, 5 SQL migration files.

**The money-critical core** is `migrations/004_royalty_hardening.sql`, which is supposed to add 5 "correct-by-construction" invariants: (1) **Conservation** — `amount = platform_fee + creator_share + grandparent_share` (CHECK); (2) **Idempotency** — UNIQUE on remix payment to block double-credit; (3) **Append-only ledger** — immutable `user_ledger`, derived balances; (4) **Multi-hop survival** — grandparent royalty survives parent GDPR erasure via snapshotted creator IDs; (5) **C2PA binding** — manifest parent must match DB parent. These existed only as ❌ GAPs at Phase-0 assessment; whether migration 004 is *actually applied to the production database* is the central unverified question of this review.
